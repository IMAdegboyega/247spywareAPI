# Migrating from GORM (SQLite/Postgres) to MongoDB

This document covers what changed and the exact steps to get the API
running again after the switch.

## What changed

- **Dropped** `gorm.io/gorm`, `gorm.io/driver/postgres`, `glebarez/sqlite`.
- **Added** `go.mongodb.org/mongo-driver`.
- **IDs** are still sequential `uint` integers. They're generated by an
  atomic `$inc` on a `counters` collection, so the frontend's existing
  numeric IDs keep working untouched.
- **Relations** are now denormalised. Each `Post` document carries an
  embedded `author` (`{id, display_name}`) and `category`
  (`{id, name, slug}`) snapshot. When a user is renamed or a category is
  renamed, the service layer sweeps the relevant posts to update those
  snapshots.
- **Editor picks** are listed with a `$lookup` aggregation that joins each
  pick row with its post in real time, so no propagation is required for
  post edits.
- **Soft deletes are gone** (GORM's `DeletedAt`). Deletes are hard.
- **Search** uses a case-insensitive regex over `title` and `content`. If
  the corpus grows, swap this for a proper Atlas Search text index.
- **`AutoMigrate` is gone.** Indexes are created at startup in
  `config.MongoDB.ensureIndexes`.

## One-time setup

### 1. Create a MongoDB Atlas cluster

1. Sign up at <https://www.mongodb.com/cloud/atlas>.
2. Create a free **M0** cluster (any region near you).
3. Under **Database Access**, add a user with read/write to any database
   and note the username + password.
4. Under **Network Access**, click **Add IP Address** and either add your
   current IP or temporarily allow `0.0.0.0/0` for development.
5. From the cluster page click **Connect â†’ Drivers â†’ Go**, copy the
   connection string. It looks like
   `mongodb+srv://USER:PASSWORD@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority`.

### 2. Configure `.env`

```bash
copy .env.example .env
```

Then edit `.env`:

```
JWT_SECRET=<something long and random>
MONGODB_URI=<the connection string from Atlas>
MONGODB_DB=247techspyware
```

### 3. Install deps + tidy go.mod

```bash
go mod tidy
```

This downloads the Mongo driver and rebuilds `go.sum`.

### 4. Seed the database

```bash
go run ./cmd/seed
```

This creates:
- the default admin (`admin@blog.com` / `admin123`)
- the five default categories (Technology, Business, Lifestyle,
  Entertainment, Sports)
- the indexes that the app relies on

You can override the admin credentials by setting `ADMIN_EMAIL`,
`ADMIN_PASSWORD`, and `ADMIN_NAME` in `.env` before seeding.

### 5. Run the server

```bash
go run ./cmd/server
```

The API listens on `http://localhost:8080`. The frontend's
`NEXT_PUBLIC_API_URL` does not need to change.

## Resetting

To wipe and start over:

1. Open the cluster in Atlas â†’ Browse Collections â†’ drop the
   `247techspyware` database.
2. Re-run `go run ./cmd/seed`.

There's no local file to delete â€” everything lives in Atlas now.

## Collections created

| Collection      | Purpose                                          |
| --------------- | ------------------------------------------------ |
| `users`         | Admins + authors                                 |
| `categories`    | Post categories                                  |
| `posts`         | Blog posts with embedded author + category       |
| `editor_picks`  | Curated featured posts (joined to posts at read) |
| `subscribers`   | Newsletter sign-ups                              |
| `adverts`       | Ad placements                                    |
| `counters`      | Atomic sequence generator (one doc per type)     |

## Troubleshooting

**`MONGODB_URI is not set`** â€” `.env` is missing or `godotenv` didn't load
it. Check that `.env` is in the same folder as where you ran `go run`.

**Auth timeout on first connection** â€” Atlas IP allowlist hasn't picked up
your IP yet. Wait 60 seconds and retry, or set `0.0.0.0/0` temporarily.

**`Failed to connect: server selection error`** â€” usually a typo in the
URI or wrong password. Re-copy the string from Atlas and remember to
URL-encode special characters in the password.

**Editor's Picks list is empty after migration** â€” the picks collection is
empty (this is a fresh DB). Add picks from the admin panel.
